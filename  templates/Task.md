## 8. Definition of Done Checklist
- [ ] Task objective met with deliverables stored in the agreed repository/location.
- [ ] Code changes peer reviewed and merged with quality gates (lint, SCA, coverage) satisfied.
- [ ] Automated and/or manual tests executed with evidence attached.
- [ ] Documentation (code comments, runbooks, playbooks, wiki) updated as required.
- [ ] Monitoring / alerting / telemetry updates implemented and validated where applicable.
- [ ] Dependencies closed or follow-up tasks logged with owners.
- [ ] Story owner/Product Owner notified of completion and acceptance confirmed.
