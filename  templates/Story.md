## 11. Definition of Ready Checklist
- [ ] Story statement and business value are clear and agreed.
- [ ] Acceptance criteria follow Gherkin format with edge cases captured.
- [ ] Designs / mockups / API contracts are attached or linked.
- [ ] Dependencies, constraints, and assumptions are documented.
- [ ] Non-functional, analytics, and compliance requirements are captured.
- [ ] Test approach and data needs are identified with QA sign-off.
- [ ] Story size estimated and prioritized with Product / Engineering alignment.

## 12. Definition of Done Checklist
- [ ] Code implemented, peer reviewed, and merged following development standards.
- [ ] Automated tests (unit, integration, UI/API) implemented and passing.
- [ ] Security / accessibility / performance checks completed where applicable.
- [ ] Documentation, runbooks, and release notes updated.
- [ ] Telemetry events validated in non-production environment.
- [ ] Feature flags configured, monitored, and rollback plan documented.
- [ ] Story validated in target environment with Product Owner sign-off.
